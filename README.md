MenuManager
Мобильное приложение для управления меню ресторана с возможностью работы в трех ролях: администратор, повар и клиент.
Описание
MenuManager - это Android-приложение, предназначенное для цифрового управления меню ресторана. Приложение позволяет администраторам управлять блюдами, повару отслеживать актуальное меню, а клиентам просматривать доступные блюда и делать заказы.
Основные функции
Для администратора (пароль: admin123):
⦁	Добавление новых блюд в меню
⦁	Редактирование существующих блюд
⦁	Удаление блюд из меню
⦁	Управление стоп-листом (добавление/удаление блюд)
⦁	Просмотр статистики по блюдам
Для повара (пароль: chef123):
⦁	Просмотр текущего меню
⦁	Просмотр стоп-листа
⦁	Отслеживание ингредиентов
Для клиента:
⦁	Просмотр доступных блюд
⦁	Поиск блюд по названию
⦁	Добавление блюд в избранное
⦁	Просмотр детальной информации о блюде (состав, цена)
⦁	Просмотр рекомендаций на основе времени суток
⦁	Сканирование QR-кода для быстрого доступа к меню
Структура приложения
Основные экраны:
1.	Выбор роли - начальный экран для выбора роли пользователя
2.	Авторизация - экран входа с паролем (кроме клиентов)
3.	Меню клиента - интерактивное меню для клиентов
4.	Панель администратора - управление меню и стоп-листом
5.	Панель повара - просмотр меню и стоп-листа
Компоненты:
⦁	DatabaseHelper - класс для работы с SQLite базой данных
⦁	MenuItem - модель данных для блюд
⦁	Ingredient - модель данных для ингредиентов
⦁	StopListItem - модель данных для стоп-листа
Технические особенности
⦁	Язык программирования: Java
⦁	База данных: SQLite
⦁	Архитектура: MVC (Model-View-Controller)
⦁	Поддержка темной и светлой темы
⦁	Адаптивный дизайн для различных размеров экранов
⦁	Анимации и переходы между экранами
Установка и запуск
1.	Откройте проект в Android Studio
2.	Синхронизируйте Gradle зависимости
3.	Запустите приложение на эмуляторе или реальном устройстве
База данных
Приложение использует локальную SQLite базу данных с автоматическим созданием структуры при первом запуске:
Таблицы:
⦁	Dish - информация о блюдах
⦁	Ingredient - ингредиенты
⦁	stopped_dishes - стоп-лист
⦁	DishIngredient - связь между блюдами и ингредиентами
Особенности использования
⦁	При первом запуске в базу данных добавляются тестовые данные
⦁	Все изменения в меню сразу отражаются для всех пользователей
⦁	Стоп-лист позволяет оперативно скрывать блюда, которые временно недоступны
⦁	Клиенты могут сохранять любимые блюда в избранное
